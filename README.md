Структура проекта (Standard Go Layout): В Go нет строгих фреймворков, навязывающих структуру, но есть общепринятый стандарт.

cmd/api/main.go — точка входа, инициализация конфигурации и запуск сервера.

internal/ — закрытый код приложения, который нельзя импортировать извне.

internal/storage — здесь будет лежать сгенерированный код sqlc.

internal/api — HTTP-хэндлеры и роутинг.

Роутинг без сторонних библиотек: Начиная с Go 1.22, стандартный пакет net/http поддерживает продвинутый роутинг (например, GET /books/{id}). Нет необходимости тащить chi, echo или gin для простого REST API.

Строгий линтинг: Использовать golangci-lint. Это поможет автоматически отлавливать проблемы со стилем, неиспользуемыми переменными и потенциальными паниками на этапе написания.

Dependency Injection: Никаких глобальных переменных для базы данных. Подключение к БД должно передаваться в хэндлеры через структуры.

Обработка ошибок: Ошибки в Go — это значения. Их нужно явно проверять (if err != nil) и возвращать осмысленные HTTP-статусы клиенту, а не просто паниковать (panic).
